# Implementation Plans

All implementation plans are saved here before any feature is built.

## Naming Convention

`[YYYY-MM-DD]-[feature-name].md`

Examples:

- `2026-03-12-f001-auth-role-access.md`
- `2026-03-12-f006-ibali-source-integration.md`
- `2026-03-12-f017-side-by-side-reader.md`

## Plan Template

Every plan must include:

1. Objective — What we are building
2. Approach — How we will build it
3. Files to create or modify — List of files
4. Steps — Numbered implementation steps
5. Acceptance criteria — How we know it is done

## Rules

- Save all plans here before coding starts
- Date them with today's date
- Use kebab-case for feature names
- One plan per feature
- Do not start coding before the plan is approved
