## Maintenance Rule

When a major project change happens:

- update `context/project-history.md` for chronology
- update `context/DECISIONS_AND_OPEN_QUESTIONS.md` for the current truth
- update `product/PROJECT_BRIEF.md` or `product/TECHNICAL_BRIEF.md` if scope changes
- update `design/` if the visual system changes
- update `engineering/` if architecture or domain assumptions change
